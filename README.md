# 聊书 · Liaoshu

> 对话式深度阅读工具 — 与作者智能体进行思想碰撞,把真实思考沉淀下来。

**MVP 交付版本 · 2026-06-11**

---

## 一、一句话定位

> **聊书** = 与作者对话式深度阅读,把每一场对话中的真实思考沉淀下来。

每个作者是一个 AI 智能体,主动带着读者思考、碰撞、生成原创洞见。

## 二、5 分钟跑起来

```bash
# 1. 安装依赖
npm install

# 2. 配置环境(已预置 .env.example,直接复制即可)
cp .env.example .env
```

第一次启动,先点击 **「+ 添加书」**,从内置书库挑一本书,开始聊。

## 六、技术亮点

### 1. 作者智能体引擎

**不是简单 RAG,而是结构化 Prompt 工程。**

每个作者智能体 = 5 个维度的配置 + 当前章节的命题 + 对话状态 → 拼装成完整 system prompt → 调用 LLM。

详见 `backend/agent-engine.js`。

### 2. 聊书笔记"只整理,不替代"

笔记生成严格遵循:
- 核心洞见 = 提取自用户原话(不 AI 替代)
- 碰撞 = 用户与作者的分歧记录
- 追问 = 用户留下的未解问题
- 延伸 = 用户的联想

详见 PRD 第 5.6 节。

### 3. 真实对话质量

冒烟测试中,作者智能体在第一轮对话中:
- 输出了 410 字的深度回应
- 引用了作者自己的思想体系
- 主动追问引导读者
- 语气贴近真实作者(柯维:温和但有力量)

## 七、API 概览

| 方法 | 路径 | 用途 |
|------|------|------|
| GET | `/api/books` | 获取内置书库 |
| GET | `/api/books/:id` | 获取单本书 |
| GET | `/api/books/:id/agent` | 获取作者智能体配置 |
| GET | `/api/user/shelf` | 获取用户书架 |
| POST | `/api/user/shelf/add` | 添加书到书架 |
| POST | `/api/user/shelf/start` | 开始聊一本书 |
| POST | `/api/user/shelf/finish` | 完成一本书 |
| GET | `/api/user/book/:id/overview` | 单书总览 |
| POST | `/api/user/chapter/start` | 开启章节(返回开场白) |
| POST | `/api/user/chapter/message` | 发送消息(LLM 调用) |
| GET | `/api/user/chapter/:book_id/:idx/dialogue` | 对话历史 |
| GET | `/api/user/chapter/:book_id/:idx/resume` | 快速 Resume |
| POST | `/api/user/chapter/close` | 收尾章节(生成笔记) |
| GET | `/api/user/chapter/:book_id/:idx/review` | 章节回顾 |
| GET | `/api/user/chapter/:book_id/:idx/note` | 章节笔记 |
| POST | `/api/user/book/:id/note/close` | 收尾整书(生成整书笔记) |
| GET | `/api/user/search?q=` | 搜索对话 |

所有用户相关接口需要 `X-User-Id` header(MVP 简化设计)。

## 八、配置说明

`.env`:

```env
PORT=3000
LLM_URL=https://aihub.firstshare.cn
LLM_KEY=sk-...                  # 你的 LLM key
LLM_MODEL=claude-sonnet-4-6     # 兼容 OpenAI 协议的模型
DATA_DIR=./data
```

`LLM_KEY` 必填。否则对话无法调用 LLM。
